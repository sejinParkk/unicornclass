# 전체 테이블 스키마

> 모든 테이블은 `lc_` prefix 사용. MariaDB 10.11, utf8mb4_unicode_ci.
> 실제 DB 기준으로 작성. 문서의 설계안과 다를 경우 이 파일이 우선.

---

## lc_admin — 관리자

| 컬럼 | 타입 | 설명 |
|------|------|------|
| admin_idx | INT UNSIGNED PK AUTO_INCREMENT | 관리자 고유키 |
| login_id | VARCHAR(50) UNIQUE NOT NULL | 로그인 아이디 |
| password | VARCHAR(255) NOT NULL | bcrypt 해시 비밀번호 |
| name | VARCHAR(50) NOT NULL | 관리자 이름 |
| email | VARCHAR(100) NULL | 관리자 이메일 |
| is_active | TINYINT(1) DEFAULT 1 | 활성 여부 |
| last_login_at | DATETIME NULL | 마지막 로그인 일시 |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | 등록일 |
| updated_at | DATETIME ON UPDATE CURRENT_TIMESTAMP | 수정일 |

> 세션 키: `$_SESSION['_admin']` = `{admin_idx, login_id, name}`
> `email` 컬럼은 `migrations/005_settings_terms.sql`에서 ALTER TABLE로 추가.

## lc_member — 회원

| 컬럼 | 타입 | 설명 |
|------|------|------|
| member_idx | INT UNSIGNED PK AUTO_INCREMENT | 회원 고유키 |
| mb_id | VARCHAR(50) UNIQUE NOT NULL | 로그인 아이디 |
| mb_password | VARCHAR(255) NULL | bcrypt 해시 (소셜 전용 NULL 허용) |
| mb_name | VARCHAR(50) NOT NULL | 이름 |
| mb_phone | VARCHAR(20) NULL | 휴대폰 번호 |
| mb_email | VARCHAR(100) NULL | 이메일 |
| signup_type | ENUM('email','kakao','naver') DEFAULT 'email' | 가입/로그인 유형 |
| social_id | VARCHAR(100) NULL | 소셜 고유 ID (카카오/네이버 user_id) |
| is_active | TINYINT(1) DEFAULT 1 | 활성 여부 (0=정지 또는 탈퇴) |
| mb_mailling | TINYINT(1) DEFAULT 0 | 메일 수신 동의 |
| mb_sms | TINYINT(1) DEFAULT 0 | SMS 수신 동의 |
| leave_at | DATETIME NULL | 탈퇴 처리 일시 (NULL=정상·정지) |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | 가입일 |
| updated_at | DATETIME ON UPDATE CURRENT_TIMESTAMP | 수정일 |

> **실제 DB 기준** (기획서와 다름):
> - PK: `member_idx` (기획서의 `mb_idx` 아님)
> - 상태: `is_active` + `leave_at` 조합 (기획서의 `mb_status` ENUM 없음)
>   - 정상: `is_active=1`
>   - 정지(admin 처리): `is_active=0, leave_at IS NULL`
>   - 탈퇴: `is_active=0, leave_at IS NOT NULL`
> - 소셜: `signup_type` ENUM (기획서의 `social_provider` 아님)
> - `mb_role`, `mb_avatar`, `last_login_at` 컬럼 없음 — 관리자는 별도 `lc_admin` 테이블
> - 세션 키: `$_SESSION['_member']` = `{member_idx, mb_id, mb_name, mb_email, signup_type, is_active}`

## lc_class — 강의

| 컬럼 | 타입 | 설명 |
|------|------|------|
| class_idx | INT UNSIGNED PK AUTO_INCREMENT | 강의 고유키 |
| category_idx | INT UNSIGNED NULL FK | 카테고리 FK (`lc_class_category`) |
| instructor_idx | INT UNSIGNED NOT NULL FK | 강사 FK |
| type | ENUM('free','premium') NOT NULL | 무료/프리미엄 |
| title | VARCHAR(200) NOT NULL | 강의명 |
| summary | VARCHAR(500) NULL | 한줄 요약 (목록 표시용) |
| description | TEXT NULL | 강의 소개 (HTML) |
| thumbnail | VARCHAR(255) NULL | 썸네일 파일명 |
| price | INT UNSIGNED DEFAULT 0 | 판매가 (원) |
| price_origin | INT UNSIGNED DEFAULT 0 | 정가 (원, 할인 전) |
| duration_days | SMALLINT DEFAULT 180 | 수강 기간 (일) |
| total_episodes | SMALLINT DEFAULT 0 | 총 챕터 수 (자동 갱신) |
| kakao_url | VARCHAR(500) NULL | 오픈채팅 URL |
| vimeo_url | VARCHAR(500) NULL | 대표 Vimeo URL |
| badge_hot | TINYINT(1) DEFAULT 0 | HOT 배지 |
| badge_new | TINYINT(1) DEFAULT 0 | NEW 배지 |
| sale_end_at | DATETIME NULL | 판매 종료일 (NULL=무기한) |
| is_active | TINYINT(1) DEFAULT 1 | 노출 여부 |
| sort_order | SMALLINT DEFAULT 0 | 정렬 순서 |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | 등록일 |
| updated_at | DATETIME ON UPDATE CURRENT_TIMESTAMP | 수정일 |

> **주의**: 기획 문서의 `class_type` → 실제 컬럼명 `type`, `discount_price` → `price_origin`.
> `total_episodes`는 챕터 추가/삭제 시 `ClassRepository::syncTotalEpisodes()`로 자동 갱신.
> 썸네일은 `storage/uploads/class/{uuid}.{ext}` 경로에 저장, `/uploads/class/{filename}` 라우트로 서빙.

## lc_class_chapter — 강의 챕터

| 컬럼 | 타입 | 설명 |
|------|------|------|
| chapter_idx | INT UNSIGNED PK AUTO_INCREMENT | 챕터 고유키 |
| class_idx | INT UNSIGNED NOT NULL FK | 강의 FK |
| title | VARCHAR(200) NOT NULL | 챕터 제목 |
| vimeo_url | VARCHAR(500) NOT NULL DEFAULT '' | Vimeo 영상 URL |
| duration | INT UNSIGNED DEFAULT 0 | 재생 시간 **(초 단위 정수)** |
| sort_order | SMALLINT DEFAULT 0 | 정렬 순서 |
| is_active | TINYINT(1) DEFAULT 1 | 노출 여부 (0=소프트 삭제) |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | 등록일 |

> **주의**: 기획 문서의 `lc_class_episode`, `vimeo_id` 컬럼은 실제로 사용하지 않음.
> `duration`은 초(seconds) 정수로 저장. UI 표시 시 `ChapterRepository::secondsToDisplay(int)` → `"M:SS"` 변환.
> 관리자 폼에서 입력 시 `"MM:SS"` 형식으로 입력 → `ChapterRepository::displayToSeconds(string)` → DB 저장.
> 챕터 저장은 강의 등록/수정 폼의 **저장 버튼** 클릭 시 일괄 처리 (`chapters_json` hidden field).

## 파일 저장 경로 요약

| 종류 | 저장 경로 | 서빙 라우트 |
|------|-----------|------------|
| 강의 썸네일 | `storage/uploads/class/{uuid}.{ext}` | `GET /uploads/class/{filename}` |
| 강의 자료 파일 | `storage/uploads/materials/{uuid}.{ext}` | `GET /uploads/materials/{filename}` (attachment) |
| 강사 프로필 사진 | `storage/uploads/instructor/{uuid}.{ext}` | `GET /uploads/instructor/{filename}` |
| 사이트 로고·파비콘 | `storage/uploads/site/{uuid}.{ext}` | `GET /uploads/site/{filename}` |
| 이벤트 배너 이미지 | `storage/uploads/banner/{uuid}.{ext}` | `GET /uploads/banner/{filename}` |
| 팝업 이미지 | `storage/uploads/popup/{uuid}.{ext}` | `GET /uploads/popup/{filename}` |

> 모든 경로는 DocumentRoot(`public/`) 외부에 위치하며, PHP 라우트를 통해서만 접근 가능.
