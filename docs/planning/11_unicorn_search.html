DESCRIPTION — 검색 결과
1
GNB
검색 후에도 GNB 유지
검색창 border #c0392b 활성 표시
2
검색 입력창 활성 상태
검색어 유지 + bold
수정 후 Enter / 🔍 → 재검색
빈칸 검색 → 전체 클래스 목록 이동
3
검색 결과 헤더
"검색어에 대한 N개의 검색 결과" 형식
클래스 수 + 강사 수 합산 표시
URL: /search?q=검색어
4
클래스 섹션
검색 로직 (OR 조건):
① 강사명 일치 → 해당 강사의 전체 클래스
lc_instructor.name LIKE '%q%' → instructor_id로 클래스 조회
② 클래스명 일치 → 해당 클래스
lc_class.title LIKE '%q%'
두 결과 합산 후 중복 제거 (UNION)
is_active = 1 필수
정렬: 관련도순 (강사명 완전일치 우선)
5
클래스 카드
기존 클래스 리스트와 동일 구조
제목 내 검색어 <mark> 하이라이트
(강사명 검색 시 강사명이 포함된 부분 하이라이트)
클릭 → /class/:id
6
강사 섹션
검색어와 강사명 LIKE 매칭된 강사 목록
lc_instructor.name LIKE '%q%'
강사명에 <mark> 하이라이트
클래스명 검색 시에도 관련 강사 노출 (클래스 결과의 instructor_id DISTINCT)
7
강사 카드
기존 강사소개 페이지와 동일 구조
이름에 <mark> 하이라이트
클릭 → /instructor/:id
i
API
GET /api/search?q=검색어
응답: { classes:[], instructors:[], total:N }

검색 로그
lc_search_log (keyword, searched_at, result_count)
결과 없는 검색어 → 관리자 콘솔 확인
추후 인기 검색어 / 자동완성 확장
결과 없음 /search?q=abcxyz 검색 결과가 없는 상태
